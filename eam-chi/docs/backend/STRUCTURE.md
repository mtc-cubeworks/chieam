# Backend Project Structure

This document outlines the architectural structure of the backend system.

## Core Philosophy: Modular & Code-First

The system is designed to be **Modular** (features are self-contained in `modules/`) and **Code-First** (Python models define the data structure, which drives the Database and JSON Metadata).

## Directory Structure (`backend/app/`)

### 1. `modules/` (The Domain Logic)

This is where 90% of your work should happen. Each feature is a self-contained module.

```text
modules/
├── core/                   # System-wide models (Auth, Workflow, Series)
├── todo/                   # Example feature module
│   ├── models/             # SQLAlchemy Models (Source of Truth)
│   │   ├── todo.py
│   │   └── todo_comment.py
│   ├── entities/           # JSON Metadata (Auto-generated by forge sync-meta)
│   │   └── todo/           # Frontend reads this
│   └── apis/               # Custom Business Logic / Validation
│       └── todo.py         # Overrides generic CRUD behavior
└── project_management/     # Another feature module
```

### 2. `core/` (System Foundation)

Infrastructure code that powers the application.

- `config.py`: Environment variables and settings.
- `database.py`: SQLAlchemy engine and session management.
- `loader.py`: Dynamic module loader (finds models in `modules/`).
- `seed.py`: Initial data population.
- `security.py`: JWT authentication and password hashing.

### 3. `meta/` (Metadata Engine)

- `registry.py`: The brain of the system. It loads entity definitions (from RAM or JSON) and serves them to the API.

### 4. `routers/` (API Endpoints)

- `entity.py`: **The Generic Router**. Handles CRUD (`/api/entity/{name}`) for _all_ entities based on metadata. You rarely need to write custom routers.
- `auth.py`: Login/Register endpoints.
- `admin.py`: System administration endpoints.

### 5. `services/` (Business Logic Services)

Reusable logic shared across the system.

- `rbac.py`: Role-Based Access Control checks.
- `workflow.py`: State machine logic.
- `naming.py`: ID generation (e.g., `WO-0001`).
- `socketio_manager.py`: Real-time updates.

### 6. `entities/` (Utilities)

- `generator.py`: Logic for `forge new-entity`.
- `__init__.py`: Metadata loading utilities.

## Key Processes

### Creating a New Feature

1.  Run `python -m app.forge new-entity` to scaffold the module.
2.  Edit `modules/{module}/models/{entity}.py`.
3.  Run `python -m app.forge sync-meta` to update JSON.
4.  Run `python -m app.forge migrate` to update DB.

### Application Startup (`main.py`)

1.  **Load Config**: Read `.env`.
2.  **Load Modules**: `core.loader` scans `app/modules` and imports all `models/*.py`.
3.  **Load Metadata**: `entities.load_all_entities` reads JSON files into the Registry.
4.  **Init DB**: SQLAlchemy connects to the database.
5.  **Start API**: FastAPI serves the application.

## Best Practices

1.  **Keep Modules Independent**: A module should ideally work even if you remove other non-core modules.
2.  **Use the Generic API**: Avoid writing custom API endpoints unless necessary. Use `FormView` and `ListView` on the frontend.
3.  **Code-First**: Always change the Python Model first, then sync the JSON.
